# HexMap 探索系统 — 架构说明

本文档为 `HexMapArchitecture.md` 的非程序向镜像。仅覆盖 HexMap 大地图系统，不含 BattleMap 战斗系统。

---

## 0. 总览

### 0.1 核心玩法循环

```
玩家在六边形地图上移动 → 消耗生存点数(AP) → 到达节点 → 触发内容
  ├─ 普通战斗 → 进入战斗关卡 → 结束后返回
  ├─ 紧急战斗 → 同上 + 时间限制
  ├─ 事件/奖励 → 弹出事件面板 → 阅读文本 → 选择选项 → 获得或失去资源
  ├─ 商店 → 弹出交易面板 → 用资源换资源/AP
  └─ 传送 → 选择目标区域 → 传送到新区起点

面板关闭后 → 刷新地图 → 检测 AP 是否为 0 或全部节点是否探索完毕 → 游戏结束
```

### 0.2 资源分层

游戏中的资源分四层管理：

```
战斗中获得 → 战斗结算加入探索资源 → 探索资源变化时自动同步到全局存档
```

- **战斗层**：战斗中采集的原始资源量
- **区域获得层**：当前区域"净赚了多少"（只记增加，不记支出）
- **探索层**：整局游戏的真实余额（含 AP）。这是唯一的数据源，所有界面读这里
- **全局层**：局外的永久资源，存盘

### 0.3 HexMap 与 BattleMap 的关系

- 进战斗：在 HexMap 上点战斗节点 → 写入关卡信息 → 跳转战斗地图
- 出战斗：战斗结算面板点确认 → AP 可能扣减 → 跳回 HexMap → 地图刷新
- 游戏结束：AP=0 或所有节点走完 → 返回主菜单（待实现）

## 1. 核心系统

### 1.1 干员总控 (URougeliteRunSubsystem)

游戏运行就存在，负责：
- 读档/存档
- 维护全局干员清单（哪些干员可用、等级多少）
- 维护全局资源
- "人才引进"事件可解锁干员（设为可参战）

Amiya 初始锁定，需通过事件解锁。

### 1.2 大地图总控 (UHexMapSubsystem)

HexMap 的核心系统，负责：
- 生成 4 个区域的 4×4 六边形节点布局
- 管理当前所在区域和坐标
- 记录 AP 消耗量和区域获得量
- 提供战斗/事件/商店/传送所需的凭证数据
- 管理修改器（事件产生的持续效果）

AP 存储在探索层资源中，初始化时从全局存档复制。AP 被改时只更新数值不触发游戏结束——游戏结束由面板关闭后的地图刷新统一检测。

### 1.3 存档与配置

- **URougeliteSaveGame**：存档容器——全局资源、干员等级、玩家等级
- **URougeliteSettings**：编辑器配置——4 张 DataTable 的引用（干员表、关卡表、图标表、事件表）

## 2. 数据层

### 2.1 区域类型

| 区域 | 主资源 |
|---|---|
| 原木 (Wood) | 原木 |
| 原石 (Rock) | 原石 |
| 稻谷 (Grain) | 稻谷 |
| 原矿 (Metal) | 原矿 |

### 2.2 节点状态

| 状态 | 含义 | 玩家能看到？ | 能点击？ |
|---|---|---|---|
| Masked (迷雾) | 未探索 | 否（被雾遮住） | 否 |
| Unmasked (可见) | 已揭示但不可达 | 是 | 否 |
| Reachable (可达) | 可移动过去 | 是 | 是 |
| Cleared (已清除) | 已去过 | 是（灰色） | 否 |

### 2.3 节点类型

| 类型 | 触发内容 |
|---|---|
| Entrance (入口) | 起点，固定在 (0,0) |
| Combat_Normal (普通战斗) | 进入普通战斗关卡 |
| Combat_Emergency (紧急战斗) | 进入紧急战斗（有时间限制） |
| Event (事件) | 弹出事件面板 |
| Reward (奖励) | 弹出奖励面板 |
| Shop (商店) | 弹出商店面板 |
| Teleport (传送) | 弹出传送面板，固定在 (3,3) |

### 2.4 事件配置结构

每个事件在 DataTable 中一行，包含：
- 事件 ID、所属区域
- 阶段数（0=单阶段）、每阶段选项数
- 每阶段一个标题、一个描述（用数组存）
- 全部选项文本列表
- 绑定的事件逻辑 C++ 类

### 2.5 节点数据结构

每个六边形地块包含：坐标、类型、状态、是否曾被到达过、内容随机种子。

新增 `bHasBeenVisited` 字段追踪节点访问历史——全部四个区域的节点都被访问过后触发游戏结束。

## 3. 节点探索

### 3.1 区域生成规则

- 每个区域 4×4 六边形棋盘
- 入口固定在 (0,0)，出口（传送点）固定在 (3,3)
- 中间 14 个节点随机洗牌后分配类型
- 紧急战斗不会出现在紧邻入口的位置（(0,1) 和 (1,0)）
- 出口的"是否访问过"初始为是（不算障碍）

### 3.2 移动规则

- 只能移动到 Reachable（可达）的节点
- 右/下方向消耗 1 AP
- 斜方向消耗 3 AP
- 修改器"无限火力"可免除消耗
- 移动时不刷新地图——刷新推迟到内容处理完毕后

### 3.3 节点揭示规则

刷新地图时：
- 当前位置的四个六边形邻居无论是可达都变为 Unmasked（揭示）
- 之前可达但没有被选择的节点保持 Unmasked（可见但不消失）
- 入口始终 Cleared
- 传送点始终 Unmasked（除非玩家站在上面）

### 3.4 区域切换

进入新区域时：先切到新区→执行修改器（旧区效果清除、下区效果激活）→旧 Cleared 节点重置为 Unmasked→刷新地图。

## 4. 事件系统

### 4.1 事件流程

1. 玩家点击事件节点→画面打开事件面板
2. 面板显示事件大图、阶段标题、描述
3. 面板下方显示当前阶段的选项按钮，点击选项→执行对应逻辑
4. 单阶段事件直接关闭；多阶段事件可自动进入下一阶段
5. 事件子类可控制"跳过某阶段"或"回到某阶段"以支持分支/环状流程

### 4.2 事件逻辑基类

每个事件对应一个 C++ 子类，在 `ExecuteOption` 中写业务逻辑：加减资源、加减 AP、创建修改器、触发战斗等。基类提供：
- 两个子系统引用（大地图总控、干员总控）
- `DetermineNextStage`——控制阶段跳转目标（-1=结束）
- `bTriggeredCombat`——若触发了战斗则阻止面板关闭（防止 AP=0 检测和 OpenLevel 竞争）

### 4.3 多阶段事件

通过 DataTable 中的 `MultiStageNumber` 和 `OptionPerStage` 配置。所有选项平铺在一个数组里，前 N 个属第一阶段，接下来 M 个属第二阶段...标题和描述也是每阶段一个。

### 4.4 事件清单

四个区域各有 3 种普通事件 + 跨区域共用的 5 种奖励事件（随机分布）。

## 5. 修改器系统

### 5.1 什么是修改器

事件给予的持续效果。有生命周期：

- **Permanent**：永久生效
- **CurrentRegion**：离开当前区域时销毁
- **NextRegion**：当前区域不生效，进入下一个区域时自动激活

### 5.2 已实现的修改器效果

| 效果 | 使用事件 |
|---|---|
| 移动不消耗 AP | 无限火力 |
| 禁止获得某资源 | 我要存钱（禁原石） |
| 资源获得倍率 | 我要存钱（下区×2）、强袭（紧急×3） |
| 全局攻击加成 | 人潮人海（+90） |
| 禁用战斗时限 | 巧击 |
| 延迟奖励 | 驰援险地（结算时给原矿）、迁徙（进入新区揭示节点） |

### 5.3 生效时机

所有修改器查询点都已经加了"是否在当前区域有效"的检查。NextRegion 修改器在旧区不会出现在检查中。

## 6. 商店与背包

### 6.1 通用图标控件 (UIcon)

商店和背包共用的图标组件。点击时广播两种信号：
- `OnIconClicked(资源类型)`：商店资源按钮、背包选择——告知"谁被点了"
- `OnIconButtonClicked`：纯点击——CostIcon 唤出背包——不关心类型

图标和名称自动从 `DT_IconImage` 表按资源类型枚举名查找。

### 6.2 商店一级界面

资源入口图标预放在 UMG 中（设定好各自的资源类型）。AP 入口单独配置。点击资源按钮→弹出二级交易面板。关闭后刷新地图。

### 6.3 商店二级交易面板

显示四个要素：CostIcon（消耗品）、TargetIcon（获得品）、输入数量框、实时计算所得框。

四种交易公式已固化在代码中。AP 兑换时 CostIcon 可点击→弹出背包选择消耗品种类。

### 6.4 背包

动态读取当前资源余额生成图标列表（不显示 AP 等特殊资源）。图标用大小固定的 SizeBox 包裹在自动换行的 WrapBox 中。

两种模式：
- 普通模式：纯浏览，ConfirmButton 隐藏
- 选择模式：商店唤出时 ConfirmButton 显示，选资源后点确认→广播回商店

---

## 7. 传送与退出

### 7.1 传送面板

四个按钮对应四个区域。当前所在区域的按钮折叠（不可传送到原地）。选中区域的按钮变大，确认后跳转新区。

### 7.2 退出面板

确认→游戏结束（ConcludeGame）。取消→关闭。

---

## 8. HexMap UI 层

### 8.1 玩家控制器

创建 HUD Widget。`HandleNodeClicked` 是核心入口：验证移动→获取内容→分发到各面板/战斗。

### 8.2 HUD Widget

大地图总画布。初始化时：折叠所有子面板→WidgetTree 扫描收集 16 个节点控件→绑定委托→为每个节点注入数据→刷新地图。面板类型为具体 C++ 类，HUD 直接调用面板方法。

### 8.3 节点控件

单个节点。图标从 `DT_IconImage` 按节点类型枚举名查表。Masked 状态显示迷雾图、按钮折叠。其他状态正常显示+StateTint 着色。Reachable 时透明按钮覆盖展开+悬停缩放。
