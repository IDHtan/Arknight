# Arknight 项目架构文档

## 0. 总览

当前项目建议从“是否属于长期保留资产”来理解，而不是只看功能。

### 0.1 核心层

1. 总状态层：`URougeliteRunSubsystem`、`URougeliteSaveGame`、`URougeliteSettings`
2. 数据描述层：`DS.h` 中的枚举与 `USTRUCT`

### 0.2 BattleMap 原型的非总状态层

1. `BattleMap/Controller`：战斗关卡驱动、输入、镜头、局内运行态控制与本场资源统计入口
2. `BattleMap`：网格地图生成与格子管理
3. `BattleMap/Cell`：地块对象
4. `BattleMap/Data`：战斗期临时数据对象
5. `BattleMap/UI`：战斗 HUD、卡片、详情、部署面板
6. `Component`：生命与索敌组件
7. `Operator`：干员与子弹
8. `Interface`：索敌/受击接口
9. `Debug`：调试入口
10. 模块入口：`Arknight.cpp`

## 1. 总状态层

### **[URougeliteRunSubsystem] (继承自 [UGameInstanceSubsystem])**

职责边界：整个项目的运行态总管。负责读取/保存存档、维护当前局外资源与干员等级缓存、装配 `GlobalRoster`，并在关卡间传递“即将进入哪张战斗地图”的凭证。

Tech Note：这是当前项目最值得保留的核心系统。即使以后不用 BattleMap，HexMap 方案也大概率仍然需要一个常驻 `UGameInstanceSubsystem` 来承接 Meta 数据、局内缓存和跨关卡跳转参数。

Properties (局外/硬盘映射):

```cpp
UPROPERTY()
URougeliteSaveGame* CurrentSaveGame; // 当前内存中的存档实例，所有资源与等级修改最终都会落到这里

UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "System|SaveLoad")
FString SaveSlotName = TEXT("MainSaveSlot"); // 存档槽名，当前固定单存档
```

Properties (局内/内存暂存):

```cpp
UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = “Roster”)
TArray<FOperatorRosterData> GlobalRoster; // 基于存档等级 + 配置表组装出来的全局干员清单

UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = “MetaProgression”)
TMap<EResourceType, int32> GlobalResource; // 局外资源缓存

UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = “System|Subsystems”)
UHexMapSubsystem* MapManager; // 指向 HexMap 运行态子系统
```

Methods (底层系统/存取流转):

```cpp
virtual void Initialize(FSubsystemCollectionBase& Collection) override; // LoadGame + CheckOperators
virtual void Deinitialize() override; // SaveGame 兜底

void LoadGame(); // 从硬盘读档；若槽位不存在则创建新的存档实例
void SaveGame(); // 覆写保存到硬盘
void CheckOperators(); // 读取 OperatorDictTable → 装配 GlobalRoster
```

Methods (局外 Meta 业务):

```cpp
void UpgradeOperator(FName OperatorName); // 提升干员等级，同步 CurrentSaveGame + GlobalRoster

// 将干员解锁为可参战（只改 bCanJoinBattle = true，不增删数组项）
// 用于”人才引进”事件
void AddOperator(FName OperatorName);
```

Methods (局内 Run 业务):

```cpp
// 写入存档 + GlobalResource
void AddResource(EResourceType Type, int32 Amount);
void AddGameResource(EResourceType Type, int32 Amount);
```

当前实现提醒：

- `CheckOperators` 末尾将 Amiya 的 `bCanJoinBattle` 设为 false，由”人才引进”解锁
- `AddOperator` 只改已有条目的 bCanJoinBattle
- `UpgradeOperator()` 当前没有自动触发 `SaveGame()`
- AP、战斗凭证、节点管理等已全部移至 `UHexMapSubsystem`

### **[URougeliteSaveGame] (继承自 [USaveGame])**

职责边界：局外永久数据容器。负责承载资源、干员等级和玩家等级。

Tech Note：这是“纯数据类”。真正的业务入口不应该直接散落在 UI 或关卡里，而应该尽量经由 `URougeliteRunSubsystem` 操作它。

Properties:

```cpp
UPROPERTY(VisibleAnywhere, BlueprintReadWrite)
TMap<EResourceType, int32> GlobalResources; // 全局资源背包，采集与结算时应读写这里

UPROPERTY(VisibleAnywhere, BlueprintReadWrite)
TMap<FName, int32> OperatorLevels; // 干员等级字典，CheckOperators/UpgradeOperator 时应读写这里

UPROPERTY(VisibleAnywhere, BlueprintReadWrite)
int32 PlayerLevel = 1; // 玩家整体账号/进度等级，当前尚未被更多逻辑使用
```

Methods:

- 无自定义方法；本类只承担序列化容器职责

### **[UHexMapSubsystem] (继承自 [UGameInstanceSubsystem])**

职责边界：HexMap 大地图运行态总控。负责节点生成、区域管理、AP 与资源追踪、战斗/事件入口凭证、修改器管理。不持有 UI 引用。

Tech Note：AP 存储在 `CurrentGameResources[AP]` 中（private）。`ConcludeGame` 两条出口均在 `RefreshAllNodeStates` 中触发（AP≤0 或全节点 visited）。`TryMoveToNode` 不调刷新——推迟到面板关闭后。

Properties (区域与节点):

```cpp
EHexRegionType CurrentRegion;          // 当前所在区域
FIntVector2 CurrentCoordinate;          // 当前所在节点坐标
TMap<EHexRegionType, FHexRegionData> AllRegionsData; // 全部 4 区域节点数据
TArray<URunModifierBase*> ActiveModifiers; // 生效中的修改器
```

Properties (跨关卡凭证):

```cpp
FName PendingBattleLevelID;              // 待进入战斗关卡 ID
EHexNodeType PendingBattleType;          // 普通/紧急，BattleGameMode 据此决定计时器
```

Properties (消耗统计):

```cpp
int32 CurrentHexMapConsumedAP;   // 当前区域累计消耗
int32 CurrentGameConsumedAP;     // 整局累计消耗
```

Properties (资源, private):

```cpp
// 区域获得量，只记正数。GetHexMapLost() 从此计算支出
TMap<EResourceType, int32> CurrentHexMapResources;

// 唯一真实余额（含 AP）。Getter 读此；Setter 同时写全局资源
TMap<EResourceType, int32> CurrentGameResources;
```

Key Methods — 探索流转:

```cpp
void InitializeNewRun(int32 Seed); // 清空→AP=全局[AP]→生成4区域
void EnterRegion(EHexRegionType);  // 先切换区域再ProcessModifiers → 重置Cleared→Unmasked → 刷新
bool TryMoveToNode(FIntVector2);   // 扣AP+移坐标，不调刷新
void RefreshAllNodeStates();       // 邻居揭示 + Reachable→Unmasked + AP检查 + visited检查
```

Key Methods — Economy:

```cpp
int32 GetResource(EResourceType) const;       // 读余额
int32 GetCurrentAP() const;                   // → GetResource(AP)
void AddGameResource(EResourceType, int32);    // 同时写 Game + 全局
void AddHexMapResource(EResourceType, int32);  // 正数写HexMap + 调AddGameResource
void ChangeCurrentAP(int32);                   // 改AP + 广播 + 累计消耗
```

Key Methods — 战斗与事件:

```cpp
void PrepareForBattle(FName LevelID, EHexNodeType NodeType); // 写两种凭证
FName GetRandomBattleLevelID(EHexNodeType) const;           // 随机选战斗关卡
void RevealAllRegionNodes();                                 // 揭示全部节点
void AddModifierInstance(URunModifierBase*);                 // 添加修改器
void ProcessRegionEndModifiers();                            // 执行→删除→升级三步
```

ConcludeGame 两条出口（均在 RefreshAllNodeStates 中）：

1. `CurrentGameResources[AP] ≤ 0` → ConcludeGame()
2. 四区域全部节点 `bHasBeenVisited` → ConcludeGame()

### **[URougeliteSettings] (继承自 [UDeveloperSettings])**

职责边界：项目级数据表与配置入口。

Tech Note：只要项目还保留“DataTable 驱动干员/关卡配置”的模式，这个类就值得继续保留。后续 HexMap 即便替换战斗对象，也仍可能继续读这里的 DataTable。

Properties:

```cpp
UPROPERTY(Config, EditAnywhere, Category = "DataTables")
TSoftObjectPtr<UDataTable> OperatorDictTable;   // FOperatorDictionary

UPROPERTY(Config, EditAnywhere, Category = "DataTables")
TSoftObjectPtr<UDataTable> LevelLayoutTable;    // FLevelLayoutConfig

UPROPERTY(Config, EditAnywhere, Category = "DataTables")
TSoftObjectPtr<UDataTable> IconImageTable;      // FIconImageRow，节点/资源图标统一查此表

UPROPERTY(Config, EditAnywhere, Category = "DataTables")
TSoftObjectPtr<UDataTable> EventConfigTable;    // FHexEventConfig，事件数据
```

关键资产读取时机：

- `OperatorDictTable`：启动后构建 `GlobalRoster` 时读取
- `LevelLayoutTable`：HexMap 筛选战斗候选池 + BattleMap 生成地图时读取
- `IconImageTable`：节点渲染、背包图标、商店图标时读取；行名必须匹配枚举短名
- `EventConfigTable`：HexMap 随机事件时读取

Methods:

- 无自定义方法；职责是“提供可编辑的配置引用”

## 2. 数据描述层

### 文件：[DS.h]

职责边界：项目的统一枚举与轻量数据结构集合，负责给总状态层、战斗层、UI 层提供一致的数据语言。

Tech Note：这是当前项目最适合继续保留的第二批内容。后续就算把 BattleMap 整体替换成 HexMap，大多数枚举和 roster/config 结构仍然可以继续用。

### 枚举

`DS.h` 当前定义了：

- `ECellType`：地块逻辑类型
- `EResourceType`：资源类型
- `EOperatorDeployType`：可部署类型
- `EOperatorAttackType`：攻击类型
- `EOperatorClassType`：职业类型
- `EDeploymentDirection`：部署朝向
- `EDamageType`：伤害类型
- `EGameSpeedState`：游戏速度状态

### **[FOperatorRosterData]**

职责边界：局外全局 roster 条目，保留“名字、类、等级、是否允许参与本场战斗”这类稳定信息。

Properties:

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
FName OperatorName;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
TSubclassOf<AOperatorBase> OperatorClass = nullptr;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
int32 OperatorLevel = 1;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
bool bCanJoinBattle = true;
```

读取建议：

- 适合在总览 UI、编队、进入战斗前复制出 `LocalRoster` 时读取
- `bCanJoinBattle` 应在“进入战斗前的编队筛选”阶段读取；`ABattlePlayerController::BeginPlay()` 当前已依赖它过滤不可参战干员

### **[FOperatorLocalRosterData]**

职责边界：局内 roster 条目，补充了实例、是否已部署、部署费用、再部署冷却等运行态信息。

Tech Note：这是 BattleMap 当前局内状态的核心结构；未来 HexMap 仍可能保留其大部分字段，但 `OperatorInstance`、朝向、部署规则等字段可能会被重新解释。

Properties:

```cpp

FName OperatorName;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
TSubclassOf<AOperatorBase> OperatorClass = nullptr;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
AOperatorBase* OperatorInstance = nullptr;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
bool bIsDeployed = false;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
EOperatorDeployType DeployType = EOperatorDeployType::Any;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
int32 OperatorLevel = 1;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
int32 CurrentDeploymentCost = 0;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
float RedeployTime = 0.0f;

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Roster")
float RedeployCooldown = 0.0f;
```

读取建议：

- 适合在战斗控制器、卡片 UI、部署面板中读取
- 不适合跨关卡长期保存；它本质上是局内临时状态

### **[FOperatorDictionary] (继承自 [FTableRowBase])**

职责边界：干员字典表行结构，用于把 `OperatorName` 映射到 `OperatorClass`。

Properties:

```cpp
UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Operator Config")
FName OperatorName;

UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Operator Config")
TSubclassOf<AOperatorBase> OperatorClass = nullptr;
```

关键资产作用：

- 它不是运行态 roster，而是“配置源”
- 只适合在 `CheckOperators()` 这类装配阶段读取

### **[FLevelLayoutConfig] (继承自 [FTableRowBase])**

职责边界：战斗地图布局配置行结构，用于描述网格尺寸和每一格生成什么地块。

Properties:

```cpp
UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Level Config")
FName LevelID;    // 关卡逻辑 ID，HexMap 按 Region 筛选候选池

UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Hex Event")
EHexRegionType TargetRegion; // 所属区域，决定该关卡归属

UPROPERTY(EditAnywhere, BlueprintReadOnly, Category="Level Config")
int32 GridX = 0;

UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Level Config")
int32 GridY = 0;

UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Level Config")
TArray<TSubclassOf<ABattleCell>> LayoutData;
```

关键资产作用：

- `LevelID` + `TargetRegion`：HexMap 按区域筛选战斗候选池，去重 `LastBattleID`
- `GridX/Y` + `LayoutData`：BattleMap 网格生成
