# 开发文档 — 数据配置与 UMG 规范

面向美术和策划。不涉及 C++ 代码编写或 UE 蓝图工作流。

---

## 一、DataTable 配置

### DT_IconImage — 图标表

**行结构：** FIconImageRow

| 列 | 类型 | 说明 |
|---|---|---|
| ResourceType | FName | 自动等于 RowName，无需手动填 |
| IconTexture | TSoftObjectPtr\<UTexture2D\> | 图标纹理 |

**RowName 命名必须严格匹配以下列表（代码通过枚举名查表）：**

| RowName | 用途 |
|---|---|
| Masked | 节点迷雾状态图标 |
| Entrance | 入口节点图标 |
| Combat_Normal | 普通战斗节点图标 |
| Combat_Emergency | 紧急战斗节点图标 |
| Event | 事件节点图标 |
| Reward | 奖励节点图标 |
| Shop | 商店节点图标 |
| Teleport | 传送节点图标 |
| Wood | 原木资源图标 |
| Rock | 原石资源图标 |
| Grain | 稻谷资源图标 |
| Metal | 原矿资源图标 |
| Food | 食物资源图标 |
| HighQualityFood | 优质口粮资源图标 |
| AP | 生存点数图标 |

**注意：** RowName 拼错会导致该图标无法显示（LogTemp Warning 会提示行未找到）。

### DT_OperatorDict — 干员字典表

**行结构：** FOperatorDictionary

| 列 | 类型 | 说明 |
|---|---|---|
| OperatorName | FName | 干员名称，RowName 与此相同 |
| OperatorClass | TSubclassOf\<AOperatorBase\> | 指向干员蓝图类 |

需包含所有干员（含 Amiya）。Amiya 初始不可参战，由"人才引进"事件解锁。

### DT_LevelLayout — 关卡布局表

**行结构：** FLevelLayoutConfig

| 列 | 类型 | 说明 |
|---|---|---|
| LevelID | FName | 关卡 ID，RowName 与此相同 |
| TargetRegion | EHexRegionType | 此关卡属于哪个区域（Wood/Rock/Grain/Metal） |
| GridX | int32 | 地图列数 |
| GridY | int32 | 地图行数 |
| LayoutData | TArray\<TSubclassOf\<ABattleCell\>\> | 逐格指定地块类型 |

每个区域至少需要几行对应该区域的关卡，否则 HexMap 筛选战斗候选池时该区域无战斗可用。

### DT_EventConfig — 事件配置表

**行结构：** FHexEventConfig

| 列 | 类型 | 说明 |
|---|---|---|
| EventID | FName | 事件 ID，RowName 与此相同 |
| TargetRegion | EHexRegionType | 所属区域 |
| MultiStageNumber | int32 | 阶段数。0=单阶段。>0=多阶段，值等于阶段数 |
| OptionPerStage | TArray\<int32\> | 每个阶段的选项数量。数组长度=MultiStageNumber |
| EventImage | TSoftObjectPtr\<UTexture2D\> | 事件插图（所有阶段共用一张图） |
| EventTitle | TArray\<FText\> | 每个阶段一个标题。长度=阶段数（或 1 用于单阶段） |
| EventDescription | TArray\<FText\> | 每个阶段一个描述。长度=阶段数 |
| Options | TArray\<FHexEventOption\> | 全部选项列表，按阶段顺序平铺 |
| LogicClass | TSubclassOf\<UHexEventLogicBase\> | 对应的事件逻辑 C++ 子类 |

**填表示例（三阶段事件）：**
- MultiStageNumber = 3
- OptionPerStage = {2, 3, 2}
- EventTitle = {"第一阶段标题", "第二阶段标题", "第三阶段标题"}
- EventDescription = {"...", "...", "..."}
- Options = [A, B, C, D, E, F, G]（按阶段顺序：前 2 个是第一阶段，接着 3 个是第二阶段，最后 2 个是第三阶段）

每个选项结构：
- `OptionTitle`：选项短标题
- `OptionDescription`：选项描述文本
- `bIsAvailable`：是否可选

## 二、UMG Widget 清单与约束

以下控件名不可修改——C++ 代码通过 `BindWidget` 按名称自动绑定。

### WBP_HexMapHUD（大地图总画布）

| 控件名 | 控件类型 | 说明 |
|---|---|---|
| MapCanvas | UCanvasPanel | 底层画布 |
| CurrentAP | UTextBlock | AP 数值显示 |
| OpenBackpackButton | UButton | 打开背包按钮 |
| ExitButton | UButton | 退出按钮 |
| InventoryPanel | UInventoryWidget | 背包面板引用 |
| EventPanel | UEventWidget | 事件面板引用 |
| ShopPanel | UShopWidget | 商店面板引用 |
| TeleportPanel | UTeleportWidget | 传送面板引用 |
| ExitPanel | UHexExitWidget | 退出面板引用 |

**布局要求：**
- 在 `MapCanvas` 下手动放置 **16 个 WBP_HexNode** 实例
- 每个实例需在 Details 面板设 `LogicalCoordinate` 为对应网格坐标（(0,0) 到 (3,3)）
- 排列方式参照六边形网格：偶数行和奇数行偏移半个身位
- 所有子面板（Inventory/Event/Shop/Teleport/Exit）默认折叠，代码在运行时控制显隐

### WBP_HexNode（节点控件）

| 控件名 | 控件类型 | 说明 |
|---|---|---|
| NodeButton | UButton | 承载点击（代码设为全透明） |
| NodeIcon | UImage | 节点图标 |

**编辑器中逐实例设置：** `LogicalCoordinate`、`HoverScale`（默认 1.15，可调）。

**美术注意：** 六边形视觉由 NodeIcon 图片承载，NodeButton 是矩形透明覆盖层。按钮点击区域为矩形——对于 4×4 网格，相邻六边形角部重叠可接受。后续如需精确六边形碰撞，由程序侧处理。

### WBP_EventPanel（事件面板）

| 控件名 | 控件类型 |
|---|---|
| EventImage | UImage |
| EventTitleText | UTextBlock |
| EventDescText | UTextBlock |
| OptionsContainer | UHorizontalBox |

编辑器设 `OptionWidgetClass = WBP_EventOption`。OptionsContainer 内不放子控件——代码动态创建。

### WBP_EventOption（事件选项按钮）

| 控件名 | 控件类型 |
|---|---|
| TitleText | UTextBlock |
| DescText | UTextBlock |
| OptionButton | UButton |

### WBP_ShopPanel（商店一级界面）

| 控件名 | 控件类型 |
|---|---|
| ExitButton | UButton |
| APButton | UIcon (WBP_Icon) |
| TransactionPanel | UShopTransactionSubPanel (WBP_TransactionPanel) |

**重要：** 在设计师中放置 6 个 WBP_Icon 实例（对应 Wood/Rock/Grain/Metal/Food/HighQualityFood），每个实例在 Details 面板设 `bHasLinkedResourceType = true` 并设对应的 `LinkedResourceType`。这 6 个控件会被代码通过 WidgetTree 扫描自动收集。不要改它们的类型或删除。

APButton 的 `bHasLinkedResourceType` 留 `false`——代码运行时动态设。

### WBP_TransactionPanel（商店二级交易确认）

| 控件名 | 控件类型 |
|---|---|
| ConfirmButton | UButton |
| ExitButton | UButton |
| CostIcon | UIcon (WBP_Icon) |
| TargetIcon | UIcon (WBP_Icon) |
| InputQuantityBox | UEditableTextBox |
| OutputQuantityText | UTextBlock |

CostIcon 和 TargetIcon 的 `bHasLinkedResourceType` 留 `false`——运行时动态设。

OutputQuantityText 放在 TargetIcon 下方，实时显示 `输入数量 ÷ 兑换比例`。

### WBP_Icon（通用图标控件）

| 控件名 | 控件类型 |
|---|---|
| IconButton | UButton |
| IconImage | UImage |
| NameText | UTextBlock |

全透明按钮。图标和名称由代码自动从 DT_IconImage 加载。可直接复用为商店/背包/结算的图标组件。

### WBP_InventoryPanel（背包面板）

| 控件名 | 控件类型 |
|---|---|
| ExitButton | UButton |
| ConfirmButton | UButton |
| IconsContainer | UWrapBox |
| ResourceIcon | UImage |
| ResourceNameText | UTextBlock |
| ResourceNumberText | UTextBlock |

编辑器中设 `IconWidgetClass = WBP_Icon`、`IconSize = (100, 100)`、`SlotPadding = (8, 8)`。

IconsContainer 不要放子控件——代码动态创建。

### WBP_TeleportPanel（传送面板）

| 控件名 | 控件类型 |
|---|---|
| TeleportButton1 | UButton |
| TeleportButton2 | UButton |
| TeleportButton3 | UButton |
| TeleportButton4 | UButton |
| ConfirmButton | UButton |

四个按钮分别对应 Wood/Rock/Metal/Grain 区域。

### WBP_ExitPanel（退出确认）

| 控件名 | 控件类型 |
|---|---|
| ConfirmButton | UButton |
| CancelButton | UButton |

### WBP_SettlementPanel（战斗结算面板）

| 控件名 | 控件类型 |
|---|---|
| IconsContainer | UWrapBox |
| VictoryText | UTextBlock |
| ConfirmButton | UButton |

编辑器中设 `IconWidgetClass = WBP_Icon`。

VictoryText 在普通战斗时自动隐藏，仅在紧急战斗时显示胜负结果。
